Shell Scripting 101

What is a Shell?

A shell is one of the two main components in the computer's operating system (OS). The other component is the kernel. Being the innermost part of the OS, the kernel facilitates communication between the hardware and software. This is what makes one OS distinct from another. In contrast, the shell is the outermost part of the OS. It is an interface that takes input in the form of a command and returns output and protects the OS from accidental damages (hence its name). To access a shell, we use a terminal.

Background

The first UNIX shell was created by Ken Thomson in 1971. It was an interactive shell, but lacked scripting ability.

Bourne Shell

> Developed by Steve Bourne
> First appeared in the Seventh Edition Bell Labs Research version of UNIX in 1977
> The popular shell BASH (Bourne Again Shell) is a derivative of the Bourne Shell
> Its command prompt is $

C Shell

> Developed by Bill Joy at UC-Berkeley in 1978
> A variation of the C Shell is the Top C Shell (TCSH)
> Suited for network/system administrators or other professionals in the field
> Its command prompt is %

Korn Shell

> Developed by David Korn at Bell Labs, released in 1983
> Has slightly more features and better performance (e.g. better print command) than Bourne Shell, but requires more effort to learn

Shell Scripting

Shell scripting is involves the writing of a script that contains a series of commands for the shell to interpret and execute.

Why learn it?

It is useful for creating a procedure that runs your own commands, as opposed to typing them into your terminal one by one. This can save you lots of time. It helps to automate some tasks of day-to-day life, such as system administration.

Drawbacks

> As much as it helps to speed things up, a shell script runs slower than a C program. However, it definitely makes life easier on the human end!
> There are many bad shell scripts out there, since it is so easy to pick up... All the more reason why you should add it to your programming tool kit :)

Tips

> Try not to complicate your script as it can hinder comprehension
> Don't Slack on your indentation game
> Be extra on achieving a goal (keep it simple!)

How to Create a Shell Script

Create a file with the .sh extension.


	      $ touch script.sh

The first line of the file should always be:


	      #! /bin/sh

(directs the script to the location of the Bourne Shell script interpreter)

How to Run a Shell Script

To run a shell script through BASH:


	      bash script.sh

To run a shell script as an executable (to be incorporated in another program): 


	      chmod a+x script.sh
	      ./script.sh

Demo 1 : Basics + Variables 


	      # Each command is on an individual line
	      echo "hello world"
	      ls
	      
	      # declare a variable
	      # NOTE: do NOT leave any space on both sides of the = sign
	      food="sushi"

Demo 2 : If Statements


	      # ---------- SYNTAX ----------
	      # if [ <condition> ]
	      # then
	      #   <command(s)>
	      # elif [ <condition> ]
	      # then
	      #   <command(s)>
	      # else
	      #   <command(s)>
	      # fi
	      # ----------------------------

	      sleep_hr=3
	      if [ "$sleep_hr" -lt "5" ]
	      then
	      echo "physically and mentally tired"
	      elif [ "$sleep_hr" -eq "5" ]
	      then
	      echo "mentally tired"
	      else
	      echo "just tired"
	      fi

Output:


	      physically and mentally tired

Demo 4 : While Loops


	      # ---------- SYNTAX ----------
	      # while [ <condition> ]
	      # do
	      #   <command(s)>
	      # done
	      # ----------------------------

	      while [ "$editor" != "sublime" ] # the spaces must exist
	      do
	      echo "emacs or vim? (type sublime to quit)"
	      read editor # take user input
	      echo "You typed: $editor"
	      if [ "$editor" = "emacs" ]
	      then
	      echo "think again"
	      elif [ "$editor" = "vim" ]
	      then
	      echo "good choice"
	      else
	      echo "never heard of that"
	      fi
	      done

Output


	      emacs or vim? (type sublime to quit)
	      >(enter vim)
	      You typed: vim
	      good choice
	      emacs or vim? (type sublime to quit)
	      >(enter sublime)
	      You typed: sublime
	      never heard of that

Demo 5 : Case Statements


	      # ---------- SYNTAX ----------
	      # case <param> in
	      #   <string1>)
	      #     <command(s)>
	      #     ;;
	      #   <string2>)
	      #     <command(s)>
	      #     ;;
	      # esac
	      # ----------------------------

	      while : # colon always evaluates to true
	      do
	      echo "carnival or renaissance?"
	      read theme
	      case $theme in
	        renaissance)
	          echo "slate says yes"
	          break
	          ;;
	        carnival)
	          echo "it's renaissance anyways"
	          break
	          ;;
	        *)
	          echo "Are you sophfrosh..."
	          ;;
	      esac
	      done

Output:


	      carnival or renaissance?
	      >(enter hello)
	      Are you sophfrosh...
	      carnival or renaissance?
	      >(enter carnival)
	      it's renaissance anyways

Demo 6 : Command Line Arguments


	      ct=0
	      
	      # $<int> accesses value of command line arguments,
	      # starting with $0, the first argument
	      # $@ is the list of all available command line arguments
	      
	      for i in $@
	      do
	      echo "arg $ct is $i"
	      ct=`expr $ct + 1` # increment ct by 1
	      done

Output:

Suppose the above code block is saved in a file named demo.sh, running


	      ./demo.sh hello there haha

would yield:


	      arg 0 is hello
	      arg 1 is there
	      arg 2 is haha

Demo 7 : Function Arguments


	      # declare a variable
	      # there are no local variables in a shell script
	      
	      g_var="original value"
	      
	      my_func() {
	      # since $@ is used in a function definition,
	      # it refers to the list of args that the function is called with
	      
	      for i in $@
	      do
	        touch "stuff/$i.txt"`
	      done
	      
	      # all var values are global
	      g_var="global var has been modified"
	      }
	      
	      my_func come watch SING!
	      ls stuff/
	      echo "g_var = $g_var"

Output:

note: assuming an empty dir named stuff/ exists, the output is


	      SING!.txt       come.txt        watch.txt
	      g_var = global var has been modified
